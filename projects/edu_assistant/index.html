Visão geral

Um copiloto doméstico conectado ao seu ecossistema local

O projeto nasceu da necessidade de operar rotina, agenda e projetos sem expor dados sensíveis a serviços externos. Todo o fluxo gira em torno da máquina do usuário.

  • Comandos por voz naturais, com transcrição rápida e precisa.
  • Respostas faladas, prontas para interação contínua e acessível.
  • Memória persistente de preferências, atividades recorrentes e arquivos importantes.
  • Base de código extensível para automações personalizadas via scripts Python.

Arquitetura

Componentização por responsabilidades

Cada pasta abriga uma camada isolada (entrada, saída, memória, dados), facilitando manutenção e extensões. 

edu_assistant/
├── main.py              # Interface voz/texto
├── config.json          # Credenciais e preferências
├── memory/              # Memórias locais, agenda e vetores
├── data/projects/       # Metadados dos seus projetos
├── modules/
│   ├── voice_input.py   # Whisper API
│   ├── voice_output.py  # Edge TTS
│   ├── gpt_client.py    # Chamadas ao modelo
│   ├── context_loader.py# Insumos de contexto
│   ├── agenda.py        # Rotina diária
│   └── actions.py       # Execução de automações
└── requirements.txt     # Dependências principais
Instalação

Configure seu assistente em minutos

Siga os passos para preparar ambiente, credenciais e iniciar o fluxo conversacional no desktop.

  1. Clone o repositório e crie o ambiente virtual:
git clone https://github.com/eduardo45MP/edu_assistant.git
cd edu_assistant
python3 -m venv venv
source venv/bin/activate  # Linux/macOS
# .\venv\Scripts\activate  # Windows
  1. Instale as dependências:
pip install -r requirements.txt
  1. Copie o arquivo de exemplo e adicione suas chaves:
cp config.example.json config.json
# Preencha com API keys, preferências e caminhos locais
  1. Execute o assistente:
python main.py

Adapte o fluxo para texto puro substituindo a captura de áudio pelo modo CLI enquanto não tiver microfone disponível.
