Overview

Built around explicit intent and human approval

The current repository already exposes a small but working local flow with clear separation between interface, orchestration, and tools.

  • Lightweight text interface for local interaction and validation.
  • Orchestrator service that classifies intent and returns traceable plans.
  • Calendar tool that answers structured queries over local sample data.
  • Draft event-creation preparation with mandatory confirmation before any real persistence.

Installation

Run locally, as a stack, or just validate the contracts

The README currently documents three practical flows: local launcher, Docker Compose, and automated checks.

  1. Create a Python 3.11+ virtual environment and install the root dependencies: 
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
  1. Run the thin local launcher:
source .venv/bin/activate
python main.py

# Open http://127.0.0.1:8000
  1. Run the full local stack with Docker Compose: 
docker compose up --build

# 8000 interface-client
# 8001 speech-stt
# 8002 orchestrator
# 8003 tool-calendar
  1. Run the current automated checks:
source .venv/bin/activate
python -m unittest discover -s tests -p "test_*.py"

# or
./scripts/test.sh

By default, the thin launcher enables the local orchestrator and calendar-tool fallbacks to keep development lightweight without the full stack.
