# Multilingual support (EN/PT)

## Overview
The portfolio ships with a lightweight bilingual system that keeps **English** and **Portuguese** as first-class languages. All public pages share the same translation runtime so language state persists across navigation and reloads. Content is expressed through translation keys in the markup and resolved at runtime based on a saved language preference.

## How it works
- **Language toggle**: every page includes a toggle button in the top-right of the header.
- **Persistence**: the selected language is stored in `localStorage` under the `language` key.
- **Markup**: translatable elements declare keys using `data-i18n` or attribute keys via `data-i18n-attr` + `data-i18n-key`.
- **Runtime**: `scripts/main.js` loads the proper dictionary for the current page and updates text, attributes, `document.title`, and `<meta name="description">`.
- **Theme label sync**: the theme button label is updated in the active language when theme or language changes.

## Page mapping
Each page declares a `data-page` key in `<body>` that maps to a translation set:
- `home`
- `edu_assistant`
- `ai_studies`
- `readmonitor`
- `russian_training`
- `nsa`

## Adding new languages
1. Add a new language dictionary under `baseTranslations` and the desired page inside `translations` in `scripts/main.js`.
2. Make sure each translatable element has a unique `data-i18n` key.
3. If you need to translate attributes (e.g., `aria-label`, meta descriptions), use `data-i18n-attr` and `data-i18n-key`.
4. Update the language toggle label texts as needed.

## Adding new pages
1. Add `data-page="<key>"` to the `<body>` tag.
2. Add translation keys for that page under `translations.<key>` in `scripts/main.js`.
3. Use the existing language toggle + theme toggle markup from other pages to keep layout consistent.
